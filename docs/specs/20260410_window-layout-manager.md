# 画面配置アプリ（Window Layout Manager）

`20260410_window-layout-manager.md`

# 背景・課題

**作業環境が3パターンあり、毎回ウィンドウを手動で配置し直すのが面倒**

- **自宅（トリプル）**：MacBook + Type-C外部モニター + HDMI外部モニター = **3画面**
- **名古屋キャンパス（デュアル）**：MacBook + 外部モニター1枚 = **2画面**（使えるとき）
- **名古屋キャンパス（シングル）**：MacBook本体のみ = **1画面**

# アプリ概要

| 項目 | 内容 |
| --- | --- |
| **アプリ名（仮）** | Window Layout Manager |
| **プラットフォーム** | macOS（デスクトップアプリ） |
| **動作イメージ** | Dockアプリ → クリックでパターン選択 → 即座にウィンドウ配置を再現 |
| **開発手段（予定）** | Claude Code 等で実装 |

# 機能一覧

### F1. ディスプレイ自動検知

- アプリ起動時（またはメニュー表示時）に、**現在接続されているディスプレイ数を自動検知**する
- 検知結果に応じて、選択可能なレイアウトパターンを動的にフィルタリングする

| 接続ディスプレイ数 | 選択可能パターン |
| --- | --- |
| **1画面**（本体のみ） | 1画面パターンのみ |
| **2画面**（本体+外部1枚） | 2画面パターン、1画面パターン |
| **3画面**（本体+外部2枚） | 3画面パターン、2画面パターン、1画面パターン |

- ディスプレイ数が足りないパターンを選択しようとした場合 → **「ディスプレイを検知できません」とエラー表示**

### F2. レイアウト保存（記憶）機能

- 現在のウィンドウ配置を**スナップショットとして保存**する
- 保存する情報（1ウィンドウあたり）：
  - アプリ名（例：Chrome、Notion、Finder など）
  - ウィンドウの位置（x, y座標）
  - ウィンドウのサイズ（幅 × 高さ）
  - 配置先のディスプレイ（どの画面にあるか）
- ウィンドウの**中身（タブやドキュメント）は管理しない**
- 同一アプリの複数ウィンドウにも対応（例：Chrome 2枚）
- パターンごとに**名前を付けて保存**できる（例：「自宅・通常作業」「キャンパス・集中」）

### F3. レイアウト再現（復元）機能

- 保存済みパターンを選択すると、**記憶したウィンドウ配置を再現**する
- 基本思想：**今開いているアプリ・ウィンドウだけを、指定された画面位置に移動する**
- 該当アプリが起動していない場合の挙動：
  - ✅ **自動起動してから配置する**
- 該当アプリのウィンドウ数が足りない場合の挙動：
  - ✅ **既存ウィンドウだけ配置する**（新しいウィンドウは開かない）
  - 保存時にChrome×2だったが現在Chrome×1の場合 → 1枚だけ配置し、残りはスキップ

### F4. パターン選択UI

- **Dockアプリ**として動作（通常のmacOSアプリ）
- アプリを開く → **現在のディスプレイ数に応じたパターン一覧を表示**
- 操作は**クリックのみ**（キーボードショートカットなし）
- ディスプレイの接続/切断時の自動切替なし（**手動選択のみ**）
- UIイメージ：

```
┌─────────────────────────┐
│  Window Layout Manager  │
│─────────────────────────│
│  🖥️ 現在のディスプレイ: 3枚  │
│─────────────────────────│
│  ▶ 3画面パターン           │
│    ├ 自宅・通常作業    [🗑] │
│    └ 自宅・会議モード  [🗑] │
│  ▶ 2画面パターン           │
│    └ キャンパス・通常   [🗑] │
│  ▶ 1画面パターン           │
│    └ シングル・集中     [🗑] │
│─────────────────────────│
│  [＋ 現在の配置を保存]      │
└─────────────────────────┘
```

### F5. パターン削除機能

- 保存済みパターンを**個別に削除**できる（🗑ボタン）
- 削除前に確認ダイアログを表示
- パターンの編集（名前変更等）は対象外（削除して再保存で対応）

# ユースケース

### UC1: 自宅で作業開始

1. MacBookにType-CとHDMIでモニター2枚を接続
2. アプリをクリック → 「3画面」の「自宅・通常作業」を選択
3. Chrome×2、Notion、ターミナル等が記憶した位置に自動配置される

### UC2: キャンパスでシングルモード

1. MacBook単体で作業
2. アプリをクリック → 「1画面」の「シングル・集中」のみ表示される
3. 選択するとMacBook画面内でウィンドウが整列される

### UC3: 新しい配置を記憶させる

1. 自分好みにウィンドウを手動で並べる
2. アプリの「＋ 現在の配置を保存」をクリック
3. パターン名（例：「自宅・開発モード」）とディスプレイ数を指定して保存

# 技術メモ

**採用方針：Swift Package Manager（Xcodeなし）**

- Swift 6.2.4（Command Line Tools経由）
- SwiftUI でUI構築
- Accessibility API（AXUIElement）でウィンドウ操作
- `swift build` でコンパイル、手動で.appバンドル化
